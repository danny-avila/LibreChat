    **Rewrite RetrievalCall.tsx to match ToolCall.tsx reference pattern:**

    1. **Remove old imports:** Remove `ProgressCircle`, `InProgressCall`, `RetrievalIcon`. These are the deprecated patterns being replaced. Keep `ProgressText` (shared), `ToolIcon` (already imported but underused), and `useProgress` (already imported).

    2. **Add new imports:**
       - `useLocalize` from `~/hooks` (for localized strings)
       - `useExpandCollapse` from `~/hooks` (for collapsible panel)
       - `useState` from `react` (for panel toggle state)

       - `OutputRenderer` from `./ToolOutput` (for output display in panel -- import only if available, otherwise use a simple `<pre>` with formatted text)

       ```typescript
       {
         initialProgress: number;
         isSubmitting: boolean;
         output?: string;
       }
       ```

    4. **Component body:**
       - `const progress = useProgress(initialProgress);` (keep existing)
       - `const localize = useLocalize();`
       - Error detection: `const errorState = isError(output);` and cancelled: `const cancelled = !isSubmitting && initialProgress < 1;`
       - Panel state: `const hasOutput = !!output && !isError(output);` and `const [showOutput, setShowOutput] = useState(false);`
       - Expand/collapse: `const { expandStyle } = useExpandCollapse(showOutput);`

    5. **Remove `renderStatusIcon` function entirely.** The three-branch icon logic (in-progress/error/completed) is an anti-pattern identified in research. Replace with a single `ToolIcon` passed as the `icon` prop to ProgressText, exactly like ToolCall.tsx does it.

    6. **JSX structure** (follow ToolCall.tsx pattern closely):
       ```tsx
       <div className="my-2.5">
         <div className="relative my-2.5 flex h-5 shrink-0 items-center gap-2.5">
           <ProgressText
             progress={progress}
             onClick={hasOutput ? () => setShowOutput(prev => !prev) : undefined}
             inProgressText={localize('com_ui_searching_files')}
             finishedText={localize('com_ui_retrieved_files')}
             errorSuffix={errorState && !cancelled ? localize('com_ui_retrieval_failed') : undefined}
             icon={<ToolIcon type="file_search" isAnimating={progress < 1 && !cancelled && !errorState} />}
             hasInput={hasOutput}
             isExpanded={showOutput}
             error={cancelled}
           />
         </div>
         {/* Collapsible output panel - only when output exists */}
         <div style={expandStyle}>
           <div className="overflow-hidden">
             {hasOutput && (
               <div className="overflow-hidden rounded-lg border border-border-light bg-surface-secondary p-3">
                 <OutputRenderer output={output} />
               </div>
             )}
           </div>
         </div>
       </div>
       ```

       Per user decision: use a SIMPLER display than ToolCallInfo. OutputRenderer handles text/JSON formatting without the parameter rendering that ToolCallInfo includes. If OutputRenderer is not suitable (check its interface), use a simple formatted text display with `<pre className="whitespace-pre-wrap text-sm text-text-secondary">{output}</pre>`.

    7. **Domain-specific text per user decision:**
       - In-progress: "Searching files..." (NOT "Running file_search" or "Searching my knowledge")
       - Completed: "Retrieved files" (NOT "Completed file_search" or "Used Retrieval")
       - Error: "Retrieval failed" (domain-specific, NOT generic "Cancelled" or "-- Failed")
       - Cancelled: handled by ProgressText's built-in `error` prop (shows CancelledIcon)

    8. **No hardcoded colors.** All colors via semantic Tailwind tokens (`border-border-light`, `bg-surface-secondary`, `text-text-secondary`, `text-text-tertiary`).

    - `"com_ui_retrieval_failed": "failed"` (error suffix matching ToolCall's `com_ui_tool_failed` pattern -- just "failed" as errorSuffix, not full sentence, because ProgressText appends it with " -- ")
    - Add keys alphabetically near existing `com_ui_` keys.

    NOTE: Check whether `com_ui_tool_failed` (existing key = "failed") can be reused for the error suffix instead of creating `com_ui_retrieval_failed`.
