## 测试场景 4: 图片路径替换测试

### 第1轮 - 各种路径格式
**操作**: 上传 `titanic.csv`
**Prompt**:
```
生成3个图表并保存。在生成图表时，请使用matplotlib保存图片到不同的路径格式，观察系统如何处理。
```

**预期结果**:
- ✅ 无论LLM生成什么格式的路径（sandbox:/, /home/user/, /tmp/等）
- ✅ 所有路径都被正确替换为 `/images/userId/timestamp-filename.png`
- ✅ 图片都能正确显示

---

### 第2轮 - 图片名称变化
**Prompt**:
```
重新生成刚才的图表，但这次给图片起不同的名字
```

**预期结果**:
- ✅ 新图片正确保存
- ✅ 新图片路径正确
- ✅ 旧图片和新图片都能访问（如果被引用）

---

### 第3轮 - 测试自动图片持久化
**Prompt**:
```
生成3个图表：柱状图、散点图和热力图。不要手动保存文件，让系统自动处理。
```

**预期结果**:
- ✅ LLM 不应调用 download_file 工具（已移除）
- ✅ execute_code 自动持久化所有图片
- ✅ 所有3个图片都正确显示
- ✅ 图片路径格式正确
- ✅ 日志显示 "Returning X image paths to LLM"

**说明**:
此测试验证 download_file 工具已被移除，所有图片持久化由 execute_code 自动处理。LLM 应直接使用 observation 中提供的 image_paths。
